# Fjell OS v0.1 ABI / Protocol Inventory

**Version:** v0.1.2.  
**Produced by:** RFC 028 (also known as RFC-v0.1.x-005).  
**Source of truth:** `crates/fjell-abi/src/syscall.rs`,
`crates/fjell-abi/src/error.rs`, `crates/fjell-syscall/src/lib.rs`.

This document enumerates every stable and unstable surface in the
v0.1.0 ABI.  It is the reference for v0.2 boundary changes and the
baseline for the v0.1.3 enforcement audit (RFC 029).

**Enforcement-status labels:**

- `enforced` — kind + rights + lease + scope are all checked.
- `partial` — some checks exist but not all (common at v0.1.x).
- `not-enforced` — no meaningful capability check.
- `development-only` — intentionally available only in debug builds.
- `deferred` — not yet implemented; placeholder in the ABI.

---

## 1. Calling Convention

Architecture: RISC-V64 `ecall`.

```
a7  = syscall number (usize)
a0–a5 = arguments (in)
a0  = SysError code on return (0 = Ok, negative = error)
a1–a5 = optional output values on Ok
sepc += 4 before sret
```

## 3. Error Codes

All errors are returned in `a0` as negative `isize` values.

| Code | Name | Meaning |
|---|---|---|
| 0 | `Ok` | Success |
| -1 | `UnknownSyscall` | Syscall number not recognised |
| -2 | `InvalidArg` | Argument is out of range or malformed |
| -3 | `PermissionDenied` | Capability missing required rights |
| -4 | `BadState` | Object is in the wrong lifecycle state |
| -5 | `InternalError` | Kernel internal error |
| -10 | `InvalidCap` | Handle is invalid or generation recycled |
| -11 | `WrongType` | Capability kind mismatch |
| -12 | `SlotEmpty` | CSpace slot is empty |
| -13 | `SlotOccupied` | Destination slot already occupied |
| -14 | `RightsExceed` | `cap_mint` granted more rights than source |
| -15 | `CapTransferDenied` | Transfer not permitted by source rights |
| -20 | `WouldBlock` | Non-blocking op would have blocked |
| -21 | `QueueFull` | Endpoint send/recv queue is full |
| -22 | `MsgTooLong` | Message exceeds the word/cap limit |
| -23 | `Canceled` | Waiting endpoint or reply was cancelled |
| -30 | `NoMemory` | Allocation failed (no physical frames) |
| -31 | `AlreadyMapped` | Virtual address already mapped |
| -32 | `NotMapped` | Virtual address is not mapped |
| -33 | `InvalidAddress` | Address is canonical-invalid or kernel-range |

**v0.2 additions:** `LeaseRevoked` (-40), `LeaseExpired` (-41),
`GenerationMismatch` (-42) per RFC 031.

## 4. Bootstrap ABI

The kernel writes a `BootInfo` struct to init's address space before
starting it, passing the virtual address in `a1` at entry.

```rust
pub struct BootInfo {
    pub version:             u32,  // 1
    pub bootstrap_cap_count: u16,
    pub cap_task_create:     u16,  // slot 0
    pub cap_derive:          u16,  // slot 1
    pub cap_audit_drain:     u16,  // slot 2
    pub cap_bootstrap_ep:    u16,  // slot 3
    pub ram_bytes:           u64,
}
```

Slot layout is fixed at v0.1.0; new capabilities are appended by
incrementing `version` and `bootstrap_cap_count`.
