# Fjell OS v0.1.0 — Known Non-Goals

**Status:** Frozen at v0.1.0 (May 2026).
**Defined by:** RFC 024 (filename `024-release-freeze-and-scope-declaration.md`,
also known as RFC-v0.1.x-001).

This document records what v0.1.0 is **not trying to be**, so that
contributors do not accidentally extend it in those directions.
The list is the non-functional counterpart to
[`v0.1.0-limitations.md`](../releases/v0.1.0-limitations.md), which
records what the system is *not capable of*.

A request to extend v0.1.0 along one of these axes is rejected by
default. Re-opening any line below requires a new RFC.

---

## Project-level non-goals (v0.1.0)

- **No new syscall** introduced after the v0.1.0 cutoff.
- **No new kernel feature** added in v0.1.x patch releases.
- **No networking of any kind** before v0.2.0 closes the security
  boundary.
- **No production secure boot** scheme adopted.
- **No TPM / DICE integration.**
- **No remote attestation protocol.**
- **No package manager.**
- **No GUI / framebuffer / windowing system.**
- **No POSIX compatibility shim.**
- **No multi-board / multi-host coordination.**
- **No distributed snapshot sync.**
- **No fleet-management plane.**
- **No SDK / public stable API.**
- **No multi-hart / SMP support.**
- **No demand paging / copy-on-write / `mmap`.**
- **No general-purpose userspace allocator** in tree.

## Scope-discipline non-goals (v0.1.x phase)

In addition to the project-level list, the v0.1.x stabilisation
phase explicitly does **not** attempt to:

- add new OS functionality (the phase is *audit + document*);
- relax existing checks for performance reasons;
- weaken the development-grade crypto for any reason;
- accept ABI breakage without an RFC.

## Re-opening a non-goal

To re-open any line above:

1. File a new RFC under `rfcs/`.
2. Reference this document under "Problem".
3. State the new milestone that absorbs the work.
4. Update this document to remove (or annotate) the relevant line.

This is the same flow `RFC 024` itself used to establish this
document.

## Relation to v0.2

v0.2.0 (*Security Boundary Closure*) does **not** lift the
project-level non-goals above — it closes enforcement gaps inside
the existing scope. The earliest version that lifts a
project-level non-goal is **v0.4.0** (minimal secure networking).
