# Fjell OS v0.1.0 — Scope

**Status:** Frozen at v0.1.0 (May 2026).
**Defined by:** RFC 024 (filename `024-release-freeze-and-scope-declaration.md`,
also known as RFC-v0.1.x-001).

This document declares **what v0.1.0 is**. It is a companion to
[`v0.1.0-limitations.md`](./v0.1.0-limitations.md), which declares
what v0.1.0 deliberately is **not**.

---

## Positioning

v0.1.0 is the **first complete local prototype** of the Fjell
architecture. It exists to make the architecture testable and
inspectable. It does not exist to be deployed as a general-purpose
operating system.

## What v0.1.0 includes

v0.1.0 ships the following capabilities, all locally verified
through QEMU smoke tests (`TEST:M1:PASS` … `TEST:M8:PASS`):

- bootable minimal kernel
- memory isolation prototype
- user tasks
- syscall path
- capability-controlled IPC prototype
- service plane prototype
- semantic stream / text proxy
- persistent state store prototype
- immutable upgrade foundation
- signed release / signed policy / immutable rootfs prototype
- local measurement chain
- local attestation record
- recovery target

Each of these is documented in the corresponding milestone's
internal design document and the v0.1.0 CHANGELOG entry.

## What "prototype" means here

For every item above:

- the data shape is real (canonical types, real records),
- the boundary types are real (capabilities, leases, endpoints),
- the *enforcement* of those boundaries is **not yet complete** —
  some checks are partial, some are debug-only, some are missing.

The gap between "exists" and "enforced" is the work of v0.2
(*Security Boundary Closure*). See the
[v0.1.x stabilization plan](../roadmap/v0.1.x-stabilization.md) and
the v0.2 RFC set (RFCs 031–043).

## Supported target

v0.1.0 supports exactly one platform configuration:

- architecture: `riscv64gc-unknown-none-elf`
- emulator: QEMU `virt` machine
- toolchain: Rust 1.91 stable + `-Z build-std`
- host: any Linux that can `apt install rustc-1.91`

No physical hardware is supported. No alternative emulator is
supported. Adding a second supported configuration is a v0.5
milestone (Multi-Platform Foundation).
