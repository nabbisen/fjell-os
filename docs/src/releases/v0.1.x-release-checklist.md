# Fjell OS v0.1.x Release Checklist

**Version:** v0.1.4.  
**Produced by:** RFC 046 (also known as RFC-v0.1.x-010).

This checklist must be completed before tagging any v0.1.x release.
Copy it into the release PR description and tick off each item.

---

## Build gates

```
☐ cargo check on the host-buildable subset passes:
    cargo check -p fjell-tools \
        -p fjell-attestation-format -p fjell-audit-format \
        -p fjell-block-format       -p fjell-config-format \
        -p fjell-device-format      -p fjell-measure-format \
        -p fjell-recovery-format    -p fjell-rootfs-format \
        -p fjell-semantic-format    -p fjell-service-api \
        -p fjell-snapshot-format    -p fjell-store-format \
        -p fjell-upgrade-format     -p fjell-verify-format

☐ fjell-kernel builds for riscv64gc-unknown-none-elf (release):
    cargo build -p fjell-kernel --target riscv64gc-unknown-none-elf \
        --release -Z build-std=core,compiler_builtins

☐ All service crates build for the target:
    cargo xtask build-services

☐ fjell-tools builds for the host:
    cargo build -p fjell-tools
```

## Release artefacts

For each v0.1.x tag, collect and record the digest of:

```
☐ kernel ELF:           target/riscv64gc-unknown-none-elf/release/fjell-kernel
☐ serial log:           saved from the release smoke test QEMU run
☐ release archive:      fjell-os-0_1_x.tar.gz (no build artefacts)
☐ release manifest:     fjell-os-0_1_x-manifest.txt (sha256sum of each artefact)
```

The archive is built by:

```bash
cargo clean
tar --exclude='target' --exclude='fjell-disk.img' \
    --exclude='tests/qemu/artifacts/*' \
    -czf fjell-os-0_1_x.tar.gz \
    -C .. workspace
```

## CHANGELOG rubric

Each v0.1.x release entry follows this structure exactly:

```markdown
## [0.1.x] - YYYY-MM-DD

### Added
### Changed
### Fixed
### Security
### Known Limitations
### Deferred to v0.2
```

`Deferred to v0.2` is the bridge to the v0.2 backlog (RFC 047).

---

## Version bump process

1. Update `workspace.package.version` in the root `Cargo.toml`.
2. Commit the version bump + CHANGELOG entry + any documentation
   updates as a single "chore: v0.1.x release" commit.
3. Tag the commit: `git tag v0.1.x`.
4. Build and verify the release archive from the tagged checkout.
