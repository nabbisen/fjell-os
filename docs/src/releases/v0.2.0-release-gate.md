# Fjell OS v0.2.0 — Security Boundary Closure Release Gate

> RFC 059 — v0.2 Security Boundary Release Gate  
> Gate token: **`TEST:V02:PASS`** — **EARNED at v0.2.14**  
> Current shipment: **v0.2.23** (post-gate warning clean-up)

---

## Summary

v0.2.0 through v0.2.8 delivered the Security Boundary Closure milestone:
every open capability-enforcement gap from the v0.1.0 audit (RFC 029, 030, 044)
was closed and 21 v0.2 QEMU negative-test markers were live.

An external static-analysis review of v0.2.8 identified 14 release blockers (RB)
and 6 high-priority findings (H).  These were addressed across 12 RFCs (048–059)
in the v0.2.9–v0.2.14 hardening line:

| Release range | RFCs | Scope |
|---|---|---|
| v0.2.9 | — | ABI/harness correction (RB-03–06, RB-13–14, H-01) |
| v0.2.10 | 048–050 | Handle-based `require_cap`; COPY/MINT/REVOKE/INSPECT rights; exact-error negative tests |
| v0.2.11 | 051–054 | Device VMA range; DMA table-full rollback; audit drain no-loss; handle-based AuditDrain |
| v0.2.12 | 055–059 | Kernel-attested IPC identity; `sys_cap_install`; bootctl and service-manager extraction |

v0.2.14 earned `TEST:V02:PASS`.  v0.2.15–v0.2.23 addressed cross-build warnings and
dead-code audits found after gate.
