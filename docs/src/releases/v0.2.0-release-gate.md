## Status (post-v0.2.8 external review)

A static-analysis review of v0.2.8 (see `dev-summary-v0_2_x.md`) identified
14 release blockers and 6 high-priority findings.  The 21/21 marker claim is
considered unreliable until the ABI/test-harness fixes (v0.2.9) are re-run.
Until then, **this gate is not granted**.

## Host-verified items (all PASS)

### Capability enforcement (RFC 031, 032) — 16 tests
`fjell-cap::enforcement::tests::*` — all `require_cap` step combinations,
`cap_drop` skips lease check.

### Lease epoch revocation (RFC 033) — 7 tests
`fjell-kernel::lease::tests::*` — O(1) epoch increment, stale-epoch rejection,
lifecycle revoke.

### Blocked IPC library (RFC 034) — 10 tests
`fjell-ipc::endpoint::tests::*` — `cancel_by_lease` matching, epoch mismatch,
empty-endpoint safety.

### Safe user copy (RFC 039) — 8 cases
Verified via standalone `rustc` + `UserPtr` unit tests: null, kernel-address,
length overflow, range-crosses-kernel, valid range.

### cap-broker policy (RFC 040) — 14 tests
`fjell-tools::policy_eval::tests::*` — BROKER-001 (default deny), BROKER-002
(deny priority), BROKER-003 (rights intersection), all service-resource pairs.

### Persistent evidence (RFC 041) — 10 tests
`fjell-snapshot-format` continuity checks, `fjell-audit-format` persist-record
roundtrip and size assertions.
