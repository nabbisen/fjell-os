### V02-S-001 (Should): Static MmioRegionTable population at boot

the grant; but the table must be populated before virtio-blk can use
the new ABI.

### V02-S-002 (Should): Semantic stream nodes for new v0.2 events

Source: RFC 041 evidence matrix. The semantic nodes should exist before
the release gate is signed; they are not individually release blockers
but collectively they are (the gate document checks the whole matrix).

### V02-S-003 (Should): v0.2 threat model update

Source: RFC 043 §Documentation gates. The threat model must be updated
for v0.2 before the release gate is signed.

### V02-S-004 (Should): v0.2 ABI inventory update

Source: RFC 043 §Documentation gates. `sys_cap_drop` (RFC 032),
`ipc_try_recv` (already exists but gap is documentation), and the
MmioMap/DmaAlloc ABI changes (RFCs 035, 036) must be reflected.

### V02-S-005 (Should): DEFERRED markers replaced by real tests

Source: RFC 042 §DEFERRED convention. All `NEG:*:DEFERRED` markers
in profiles must be replaced by real `:PASS` tests before v0.2.0 tag.

### V02-S-006 (Should): DmaRegion lease binding integrates with lifecycle revoke

Source: RFC 036 + RFC 033 §2.10. Task-exit lifecycle revoke must
trigger DmaRegion zeroize, not only explicit revoke.

### V02-S-007 (Should): verifyd, rootfsd, snapshotd extracted from inline-init

Source: RFC 038 (extraction order: storaged, bootctl, verifyd, upgraded,
rootfsd, snapshotd). Items 3–6 are Should; items 1–2 are Release Blocker.

### V02-S-008 (Should): audit IPC denied and IPC sent events added

Source: RFC 044 matrix — IPC denied: no channel. Needed before the
evidence matrix target state is satisfied.

### V02-S-009 (Should): snapshot includes audit_last_seq and dropped_count

Source: RFC 041 §Snapshot inclusion.

### V02-S-010 (Should): RFC 029 enforcement audit updated post-implementation

Source: RFC 029 §9 checklist. After every RFC in Epics A–C lands, the
audit document's "Implemented?" column must be updated.

### V02-C-001 (Could): Subscription-based audit cursors

Source: RFC 039 (global drain cursor; per-subscriber deferred). A second
auditd subscriber would need this; not needed for v0.2.0.

### V02-C-002 (Could): DmaShare capability path

Source: RFC 029 §6 (dma_share classification: Missing). No v0.2 use case
requires DmaShare; the call site is a stub.
