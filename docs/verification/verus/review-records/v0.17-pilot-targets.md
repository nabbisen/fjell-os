### Two real findings from the machine-check (not visible to point/property tests)

1. **`capability`: two lemmas needed `by(bit_vector)`.** `zero_is_subset` and
   `equal_rights_allowed` assert universally-quantified bitwise facts
   (`0 & !parent == 0`, `parent & !parent == 0`). The conformance and property
   tests pass because they evaluate concrete values, but the SMT solver does
   not discharge these over all `u64` without the bit-vector solver. Added the
   hint; both now verify. (`subset_is_transitive` already used it.)
2. **`verus-check` xtask invoked Verus without `--crate-type=lib`.** Proof
   modules are library files with no `main`, so the bare `verus <file>` call
   failed with `E0601`. Fixed `run_verus` to pass `--crate-type=lib`; the
   xtask now reports PASS correctly.

Both fixes are recorded in the v0.17.1 CHANGELOG entry. The proofs remain
1:1-mapped to shipped Rust (`is_subset_of`, lease epoch logic,
`select_bcb_mirror`) and the 19 conformance + 13 property tests still pass.

The proofs are additionally:

## Invariant Review

```text
capability:
  subset(child,parent) := (child & !parent)==0  — matches CapRights::is_subset_of byte-for-byte.
  Lemmas: no-amplification, zero⊆any, reflexive, transitive, (property) antisymmetric.
  Manual: all hold for bitset subset; transitivity discharged in Verus by bit_vector.
  No behavior accidentally excluded: predicate is total over u64×u64.

lease:
  usable := active && epoch==issue  — matches check_active.
  revoke := {active:false, epoch+1} — matches revoke() (kernel uses wrapping_add u32).
  Lemmas: revoked-not-usable, inactive-rejects-any, revoke-advances-epoch.
  Manual: revoke sets active=false ⇒ usable() false regardless of epoch — holds.
  Caveat: model uses unbounded nat; kernel wraps at 2^32 revokes of one lease
          (documented in RFC-v0.17-003 conformance note; not reachable in practice).

boot-control:
  select matches select_bcb_mirror's four outcomes exactly.
  Lemmas: none-iff-both-invalid, valid-beats-invalid, higher-gen-wins,
          equal-gen-tiebreak, totality.
  Manual: case split is exhaustive and mutually exclusive — holds.
```
