## Machine-check status (honest)

The Verus binary cannot be installed in the build sandbox: GitHub
release-asset hosts (`objects.githubusercontent.com`,
`release-assets.githubusercontent.com`) return `host_not_allowed`, and
source builds need the same blocked infrastructure plus z3. The proofs are
therefore **not machine-checked here**. They are:

1. **Mapped 1:1** to shipped Rust predicates (`is_subset_of`, lease epoch
   logic, `select_bcb_mirror`).
2. **Point-tested** by conformance tests (19 cases).
3. **Property-tested** over random inputs for each proved lemma
   (`fjell-proptest/tests/verus_lemma_properties.rs`, 13 properties).
4. **Manually reviewed** below, obligation by obligation.

A developer with network access machine-checks via `verification/verus/TOOLCHAIN.md`.

## Invariant Review

```text
capability:
  subset(child,parent) := (child & !parent)==0  — matches CapRights::is_subset_of byte-for-byte.
  Lemmas: no-amplification, zero⊆any, reflexive, transitive, (property) antisymmetric.
  Manual: all hold for bitset subset; transitivity discharged in Verus by bit_vector.
  No behavior accidentally excluded: predicate is total over u64×u64.

lease:
  usable := active && epoch==issue  — matches check_active.
  revoke := {active:false, epoch+1} — matches revoke() (kernel uses wrapping_add u32).
  Lemmas: revoked-not-usable, inactive-rejects-any, revoke-advances-epoch.
  Manual: revoke sets active=false ⇒ usable() false regardless of epoch — holds.
  Caveat: model uses unbounded nat; kernel wraps at 2^32 revokes of one lease
          (documented in RFC-v0.17-003 conformance note; not reachable in practice).

boot-control:
  select matches select_bcb_mirror's four outcomes exactly.
  Lemmas: none-iff-both-invalid, valid-beats-invalid, higher-gen-wins,
          equal-gen-tiebreak, totality.
  Manual: case split is exhaustive and mutually exclusive — holds.
```

## Assumptions

```text
Assumption 1: validity (magic+CRC) is established before mirror selection;
              CRC is out of scope for the first boot-control model.
Assumption 2: lease epoch is monotonic; wraparound needs 2^32 revokes (unreachable).
Assumption 3: capability rights are a u64 bitset (matches CapRights(pub u64)).
```
