# RFC-v0.6-003: Semantic Schema Compatibility and Format Fuzzing

**Status.** Implemented (v0.6.0)

## Status

Draft (revised, supersedes pack v0.6-003 draft)

## Target Version

`v0.6.0`.

## Phase

Verification, Fuzzing, and Property Testing — Epic C (Schema / Fuzz).

## Related Work

- v0.5 RFC 004 — semantic catalog v1.
- v0.3 RFCs 002/004 — keyring + attestation formats (fuzz targets).
- v0.4 RFC 004 — UpdateIndex JSON-subset parser (fuzz target).

---

## 1. Summary

Stand up a continuous fuzzing harness that targets every parser in Fjell's
user-space surface, with two specific goals:

- **never panic on adversarial input**;
- **never accept input that does not round-trip through serialise →
  parse → serialise**.

Add a *schema compatibility* gate that runs against every PR: for each
public format (`AttestationRecordV2`, `ReleaseMetadata`, `KeyringSnapshot`,
`BoardProfile`, `UpdateIndex`, `DiagnosticBundle`, semantic-v1 records),
the gate compares the canonical-digest formula against the previous git
tag and fails on any change unless an explicit `BREAKING` marker is in the
commit message.

---

## 2. Motivation

Parser hardening lags feature work. By v0.6, Fjell has ~10 user-facing
parsers. None has yet been continuously fuzzed. v0.6 is the point to
introduce fuzzing as part of CI rather than a one-shot exercise.

Schema compatibility is an even subtler bug: changing the canonical digest
formula in a non-breaking PR silently invalidates every persisted record
in the field. The compatibility gate makes this visible.

---

## 3. Goals

```text
- Continuous fuzz targets for: AttestationRecordV2, ReleaseMetadata,
  RollbackRecord, KeyringSnapshot, BoardProfile, UpdateIndex,
  DiagnosticBundle, semantic-v1 records.
- Each target: parse-never-panic + round-trip property.
- Schema-compatibility gate runs in CI on every PR.
- Corpus seeds shared via the repo under `fuzz/corpora/`.
- Sanitizers (UB / Memory) enabled where possible on host.
```

## 4. Non-Goals

```text
- No kernel-side fuzzing.
- No coverage-guided fuzzing of QEMU images.
- No fuzz time budget beyond CI's per-PR ceiling (5 min/target).
```

## 5. External Design

### 5.1 Targets

```text
fuzz/
  fuzz_targets/
    attestation_v2_parse.rs
    release_metadata_parse.rs
    rollback_record_parse.rs
    keyring_snapshot_parse.rs
    board_profile_parse.rs
    update_index_parse.rs
    diagnostic_bundle_parse.rs
    semantic_record_parse.rs
  corpora/
    attestation_v2/
    release_metadata/
    rollback_record/
    keyring_snapshot/
    board_profile/
    update_index/
    diagnostic_bundle/
    semantic_record/
```

Each target uses cargo-fuzz's `libfuzzer-sys` driver:

```rust
#![no_main]
use libfuzzer_sys::fuzz_target;
fuzz_target!(|data: &[u8]| {
    if let Ok(rec) = fjell_attestation_format::parse_v2(data) {
        let mut out = [0u8; 1024];
        let n = rec.serialise(&mut out).unwrap();
        let rec2 = fjell_attestation_format::parse_v2(&out[..n]).unwrap();
        assert_eq!(rec, rec2);
    }
});
```

### 5.2 Schema-compatibility gate

Each public format has a "frozen digest schema" file:

```text
crates/fjell-attestation-format/schema/v2.frozen
crates/fjell-upgrade-format/schema/release-metadata-v1.frozen
crates/fjell-upgrade-format/schema/rollback-record-v1.frozen
crates/fjell-keyring/schema/snapshot-v1.frozen
crates/fjell-platform-format/schema/board-v1.frozen
crates/fjell-update-index/schema/index-v1.frozen
crates/fjell-diag-format/schema/bundle-v1.frozen
crates/fjell-semantic-v1/schema/intent-v1.frozen
```

The frozen file lists, in canonical order, every byte's contribution to
the digest. CI generates the current schema from source and compares
against the frozen file:

```text
$ fjell-tools schema dump --crate fjell-attestation-format --format v2
$ diff -u  crates/fjell-attestation-format/schema/v2.frozen  /tmp/v2.now
```

Difference fails CI unless the commit body contains
`BREAKING-SCHEMA: <crate>::<format>` and a matching ADR is touched.

## 6. Data Model

### 6.1 Frozen file format

```text
# generated by fjell-tools schema dump
# crate: fjell-attestation-format
# format: AttestationRecordV2
domain "FJELL-ATTEST-V2"
field schema_version            u16 LE
field record_id                  u8 [8]
field created_tick               u64 LE
field profile                    u8
field provider.provider_id       u32 LE
field provider.kind              u8
field provider.profile_tag       u8
field provider.gen               u16 LE
...
end
```

The format is line-based, comment-tolerant, order-significant.

### 6.2 BREAKING contract

A "BREAKING-SCHEMA" change in the commit body must also:

- bump the format's `schema_version` constant;
- add an ADR entry under `docs/src/adr/`;
- update `frozen` file in the same commit.

CI's gate enforces all three.

## 7. Internal Design

### 7.1 cargo-fuzz integration

A new top-level `fuzz/` directory using `cargo +nightly fuzz`. Targets
build only with nightly because libfuzzer requires `-C
instrument-coverage`.

Per-target run:

```bash
cargo +nightly fuzz run attestation_v2_parse -- -max_total_time=300
```

Each CI build runs all targets for 300 s each.

### 7.2 Sanitizer policy

```text
- ASAN: enabled on Linux (UB sanitizer + Address sanitizer).
- TSAN: not used (single-thread parsers).
- MSAN: not used (rustc doesn't currently support it on no_std-friendly
  crates).
```

### 7.3 Corpus management

- Seed corpus is checked-in; CI also persists the *post-run* corpus to a
  scratch artefact and merges new coverage-improving inputs back to the
  repo as a follow-up PR.
- Each crash discovered becomes a checked-in regression test under
  `tests/fuzz_regressions/`.

### 7.4 Round-trip property

```rust
fn round_trip<T: Serialise + Parse + Eq>(bytes: &[u8]) {
    if let Ok(t) = T::parse(bytes) {
        let mut out = [0u8; T::MAX_SIZE];
        let n = t.serialise(&mut out).unwrap();
        let t2 = T::parse(&out[..n]).unwrap();
        assert_eq!(t, t2);
    }
}
```

A parser may legitimately reject malformed inputs (`Err(_)`); the
property only constrains the accepted-input set.

## 8. Security Design

### 8.1 What this RFC catches

```text
- Memory-safety bugs in parsers (caught by ASAN).
- Integer-overflow or panic on adversarial inputs.
- Schema drift that silently breaks persisted state.
- Round-trip non-injectivity (parser accepts a malformed but "close"
  input that re-serialises to a different canonical form).
```

### 8.2 Audit emission

None at runtime. CI failures are surfaced as job logs and PR comments.

---

## 9. Memory / Resource Design

- CI uses ~300 MiB resident per fuzz target during runs.
- Corpus storage in repo: bounded to 5 MiB per format; older inputs
  pruned by post-merge job.

---

## 10. Compatibility and Migration

- No runtime change.
- New CI matrix entry `fuzz` (nightly Rust).
- `BREAKING-SCHEMA` policy documented in CONTRIBUTING.md.

## 11. Test Strategy

### 11.1 Harness self-tests

```text
- frozen_files_present_for_every_format
- schema_dump_matches_frozen_baseline
- breaking_commit_without_adr_fails_check
- breaking_commit_without_schema_bump_fails_check
- corpus_directory_exists_for_every_target
```

### 11.2 Per-target acceptance

```text
- 5 minutes of fuzzing produces zero crashes.
- Round-trip property holds for all accepted inputs.
- Coverage exceeds a per-target threshold (e.g., 70% line, 60% branch).
```

## 12. Acceptance Criteria

```text
- fuzz/ tree exists with 8 targets.
- frozen schemas exist for all 8 formats.
- CI runs fuzz nightly + schema-compat per-PR.
- Initial CI run is clean (or has discovered bugs fixed before merge).
- ADR-v0.6-003 filed.
```
